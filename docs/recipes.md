# Recipes — record, edit, replay

A **recipe** is the playback artifact Ponder produces from any flow you drive. It's a `<id>.recipe.ts` + `<id>.json` pair under `~/.ponder/recipes/`. The `.recipe.ts` is hand-editable; the `.json` is the deterministic manifest that the replay engine consumes.

## Recording

Every browser_* / screen_* / agent_do tool call gets appended to a single process-wide trace buffer. To snapshot it into a recipe:

```
ponder_recipe_start({ task: "Search Marketplace for bulbasaur" })   // optional: clears the buffer
browser_navigate({ url: "https://www.facebook.com/marketplace" })
browser_snapshot()
browser_type({ ref: "e12", text: "bulbasaur", submit: true })
ponder_recipe_save({ task: "Search Marketplace for bulbasaur" })
```

`ponder_recipe_save` returns the saved id + a `\`\`\`ts ... \`\`\`` block with the generated code so the orchestrator can echo it inline.

`ponder_recipe_start` is optional — the buffer is rolling. Pass `fromIndex` to `ponder_recipe_save` to slice just the tail of a longer trace.

## Anatomy of a `.recipe.ts`

```ts
/**
 * Ponder recipe — generated by recording.
 *
 * Task:     Search Marketplace for bulbasaur
 * Started:  2026-05-12T18:30:00.000Z
 * Outcome:  done
 * Duration: 4.2s
 * Steps:    3
 */

import { defineRecipe } from "ponder";

export default defineRecipe({
  task: "Search Marketplace for bulbasaur",
  async run({ page }) {
    // browser_navigate  (+0.0s)
    await page.goto("https://www.facebook.com/marketplace");
    // browser_snapshot  (+1.1s)
    // (snapshot is implicit — the recorder captures it for refLabel resolution)
    // type "bulbasaur" into the search box  (+2.3s)
    await page.getByRole("textbox", { name: "Search Marketplace" }).fill("bulbasaur");
    await page.getByRole("textbox", { name: "Search Marketplace" }).press("Enter");
  },
});
```

- The body of `run()` is **raw Playwright**. You can rip it out and paste it into any Playwright test suite — no Ponder-specific wrappers inside.
- `screen.*` helpers appear for OS-level actions (Finder rows, native dialogs, Spotlight). They re-ground via the vision model against a fresh screenshot using the recorded natural-language target.

## Replay

From the CLI:

```bash
ponder run <id>                     # replay latest
ponder run <id> --reground          # re-ground OS-level clicks against the live screen
ponder run <id> --step 5            # skip the preamble
ponder run <id> --watch             # edit-and-replay loop
ponder run <id> --dry               # print steps without executing
```

From the SDK:

```ts
import { loadRecipe, replayRecipe } from "ponder";

const recipe = await loadRecipe("2026-05-12_18-30-00-search-marketplace");
const result = await replayRecipe(recipe!, { reground: true });
console.log(result); // { ok: 3, failed: 0, durationMs: 4321 }
```

From the HTTP bridge:

```ts
await client.recipe.run("2026-05-12_18-30-00-search-marketplace", { reground: true });
```

## When replay fails

The replay engine halts on first failure and writes `<id>.last-failure.png` next to the recipe. Open the `.recipe.ts` in your editor (or `ponder open <id>`), fix the bad step, then re-run.

Browser ref-based steps (`browser_click`, `browser_type`) use Playwright's `role`+`name` selectors and survive page reloads. OS-level coordinate clicks (Finder, Spotlight) need `--reground` for any cross-session replay because pixel coordinates rarely survive.

## Storage layout

```
~/.ponder/recipes/
  2026-05-12_18-30-00-search-marketplace.json    # manifest (replay consumes this)
  2026-05-12_18-30-00-search-marketplace.recipe.ts # editable code (you edit this)
```

`ponder where` prints the directory path. `ponder list` shows everything newest-first.

Legacy directories (`~/.ponder/sessions/`, `~/.holo3-agent/sessions/`) are migrated on first read.
